# 第5章 网络层：控制平面 (Network Layer: Control Plane)

> **章节定位**：聚焦网络层控制平面——负责确定分组从源到目标的路径，涵盖路由算法原理、自治系统内外路由协议、SDN新型控制架构及网络管理协议，与第4章数据平面互为补充、共同构成完整的网络层功能体系。

---

## 📋 章节导航与重难点

### 5.1 [导论](./5.1_Introduction.md)
- **核心**：
  - 转发(Forwarding)与路由(Routing)的本质区别：转发是局部逐跳操作，路由是全局路径计算
  - 控制平面的两种实现范式：传统分布式路由(Per-router)与SDN逻辑集中式控制
  - SDN架构核心：远程控制器通过本地控制代理(CA)与交换机交互，实现灵活的网络编程
- **难点**：
  - 区分"逻辑集中"与"物理集中"：SDN控制器可分布式部署但保持统一逻辑视图
  - 理解转发(数据平面)与路由(控制平面)在功能边界上的清晰划分

### 5.2 [路由选择算法](./5.2_Routing_Algorithms.md)
- **核心**：
  - 图论抽象：路由器作为节点，链路作为边，代价(cost)作为权重
  - 链路状态(LS)算法：泛洪获取全局拓扑 + Dijkstra最短路径计算
  - 距离向量(DV)算法：Bellman-Ford方程迭代 + 局部邻居信息交换
  - LS与DV的权衡：消息复杂度、收敛速度、健壮性的三维对比
- **难点**：
  - Dijkstra算法的临时/永久节点标记过程，迭代更新容易混淆
  - DV的无穷计算问题(Count-to-Infinity)：坏消息传播慢的分布式本质
  - 水平分裂(Split Horizon)及其失效场景：环路拓扑中的相互依赖陷阱

### 5.3 [自治系统内部路由](./5.3_Intra_AS_Routing.md)
- **核心**：
  - RIP协议：基于DV的跳数度量，最大15跳限制，30秒周期性交换，UDP封装
  - OSPF协议：基于LS的开放标准，IP直接封装，支持认证/多路径/多TOS
  - 层次化OSPF：区域(Area)划分降低规模复杂度，骨干区域(Backbone)互联
  - 路由器角色：内部路由器、区域边界路由器、骨干路由器、边界路由器
- **难点**：
  - RIP跳数限制(16=∞)的设计约束及其对网络规模的限制
  - OSPF区域间信息隐藏：内部路由器如何仅知方向而不知完整拓扑
  - 区域边界路由器(ABR)的距离汇总机制与具体实现

### 5.4 [ISP之间路由：BGP](./5.4_Inter_AS_Routing_BGP.md)
- **核心**：
  - BGP地位：互联网的"胶水协议"，事实上的AS间路由标准
  - 路径矢量(Path Vector)机制：携带完整AS路径而非仅距离，天然防环
  - eBGP与iBGP的分工：eBGP获取跨AS路由，iBGP在AS内部扩散
  - 策略路由：基于商业关系(对等/客户-提供商)的路由过滤与选择
  - BGP报文：OPEN/UPDATE/KEEPALIVE/NOTIFICATION的交互流程
- **难点**：
  - BGP的策略属性：理解路由选择并非纯技术最优，而是商业策略驱动
  - AS-PATH的多重作用：既是环路检测工具，也是策略决策依据(避免特定AS)
  - NEXT-HOP的跨AS语义：从当前AS到下一跳AS的多条物理链路选择
  - iBGP全互联的扩展性问题：大型AS中需路由反射器降低复杂度

### 5.5 [SDN控制平面](./5.5_SDN_Control_Plane.md)
- **核心**：
  - SDN核心思想：控制平面与数据平面分离，逻辑集中控制
  - OpenFlow协议：控制器与交换机之间的标准南向接口，"匹配+动作"范式
  - 现代控制器：OpenDaylight(ODL)、ONOS等提供网络编程API
  - 应用场景：网络虚拟化、流量工程、动态策略部署
- **难点**：
  - SDN与传统路由的范式对比：通用转发(匹配+动作) vs 基于目的地的转发
  - 控制器的可靠性与可用性：逻辑集中带来的单点故障风险及分布式备份

### 5.6 [ICMP协议](./5.6_ICMP.md)
- **核心**：
  - ICMP功能定位：IP协议的伴随协议，负责差错报告与诊断
  - Ping工具：回声请求(Echo Request)与应答(Echo Reply)测试可达性
  - Traceroute原理：利用TTL超时或端口不可达探测逐跳路径
  - 报文封装：直接承载于IP数据报中，协议字段标识为ICMP
- **难点**：
  - ICMP与IP的层次关系：作为IP的一部分使用IP传输，不独立于IP之上
  - Traceroute的两种实现路径：UDP端口不可达 vs ICMP回声应答

### 5.7 [网络管理与SNMP](./5.7_Network_Management_and_SNMP.md)
- **核心**：
  - SNMP架构：管理站(NMS)、代理(Agent)、MIB(管理信息库)三层结构
  - 管理功能：设备状态监控、性能统计收集、配置变更管理
  - SMI标准：管理信息的语法与语义定义规范
  - SNMPv3增强：认证、加密、访问控制的安全机制升级
- **难点**：
  - MIB的ASN.1表示法：抽象语法标记的编码复杂性
  - SNMPv3与历史版本的兼容性：安全增强与存量设备的共存
