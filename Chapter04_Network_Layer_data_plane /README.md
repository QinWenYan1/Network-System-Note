## 🎯 核心概念速览

### 网络层两大核心功能
- **转发 (Forwarding)** — 本地路由器功能：分组从输入端口到输出端口的本地动作
- **路由 (Routing)** — 网络范围逻辑：决定分组从源到目的的端到端路径（控制平面内容，见第5章）

### 关键机制
| 机制 | 作用 |
|------|------|
| 最长前缀匹配 (Longest Prefix Matching) | 转发表查找时使用TCAM硬件实现的高速检索，匹配最长的IP前缀 |
| 分片与重组 (Fragmentation & Reassembly) | 大数据报在不同MTU链路上分片传输，仅在最终目的主机重组 |
| NAT (Network Address Translation) | 私有网络共享少量公网IP，通过端口映射实现多主机同时上网 |
| CIDR (Classless Inter-Domain Routing) | 无类域间路由，子网部分可在任意位置，格式 a.b.c.d/x |

## 📝 PPT 章节总结

### 一、网络层功能回顾
- **转发 (Forwarding)**：对于从某个端口到来的分组，转发到合适的输出端口 —— 类比：一个多岔路口的进入和转出过程
- **路由 (Routing)**：决定分组从源端到目标端的路径 —— 类比：规划从源到目标的旅行路径

### 二、数据平面 vs 控制平面
- **数据平面**：本地的、每个路由器的功能；决定某个从某个端口进入的分组从哪个端口输出
- **控制平面**：网络范围的逻辑；决定分组端到端穿行于各个路由器的路径

### 三、两种控制平面实现方式
| 方式 | 特点 |
|:---|:---|
| 传统 Per-Router 控制平面 | 每台设备上既实现控制功能、又实现数据平面；控制功能分布式实现 |
| SDN（逻辑集中的控制平面） | 远程控制器与本地控制代理（CAs）交互 |

### 四、路由器组成
- **路由功能**（控制平面，软件）：运行路由选择算法/协议，生成路由表
- **转发功能**（数据平面，硬件）：根据路由表进行分组转发，纳秒级速度
- **关键组件**：输入端口（查找转发表）→ 交换结构（内存/总线/互联网络）→ 输出端口（缓存与调度）

### 五、IP协议
- **数据报格式**：20字节头部（版本、头部长度、TOS、总长、标识、标志、片偏移、TTL、上层协议、校验和、源/目的地址）
- **分片**：因不同链路MTU不同，大数据报需分片；重组只在最终目的主机进行
- **IPv4编址**：32位地址，分类编址（A/B/C/D/E类），CIDR无类编址 a.b.c.d/x
- **NAT**：私有地址 ↔ 公网地址转换，利用16位端口字段支持6万+同时连接
- **IPv6**：128位地址，固定40字节头部，不允许分片，移除头部校验和

### 六、通用转发和SDN
- **匹配 (Match)**：基于多个头部字段（不仅是目的IP）
- **行动 (Action)**：转发、丢弃、修改字段等
- **OpenFlow**：流表实现通用转发的具体实例

## 📚 重点难点标记

🔴 **核心重点**
- 路由器内部结构：输入端口处理 → 交换结构 → 输出端口调度

- HOL（Head-of-the-Line）阻塞的产生原因
- 三种交换结构（内存/总线/互联网络）的性能差异
- IP分片与重组的具体过程
- NAT穿越的技术挑战
- SDN与传统路由的架构差异
